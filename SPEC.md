# アクスタ入稿データ作成ウェブアプリ 仕様書

**仮称：AcSta Studio（アクスタスタジオ）**
**Version 1.0 / 2026-06-11**

---

## 1. 概要

アクリルスタンド（アクスタ）の印刷入稿データを、ブラウザ上で半自動生成するウェブアプリ。
透過PNGを配置するだけで、カットライン生成・白版生成・間隔チェックを自動で行い、最終的に**レイヤー分けされたPDF**として書き出す。

### 解決する課題
従来Illustratorで手作業していた以下の工程を自動化する：
- 画像輪郭から0.5mm離したカットラインの作成（鋭角を丸める処理込み）
- カラー版から2px縮小した白版の作成
- オブジェクト同士の間隔（4mm）確認
- 3レイヤー構成での入稿ファイル作成

### 非ゴール（v1ではやらないこと）
- 自動ネスティング（最適配置の自動計算）— 配置は手動、間隔チェックのみ自動
- .ai形式での直接書き出し — レイヤー付きPDFで代替
- イラスト自体の編集・背景透過処理 — 透過済みPNGの入力を前提とする

## 2. 技術スタック

| 領域 | 技術 | 用途 |
|---|---|---|
| ホスティング | GitHub Pages | 完全静的・サーバー不要 |
| フレームワーク | React + Vite（または Vanilla + Vite） | SPA |
| キャンバス | SVG（DOM） | 編集画面。各オブジェクトを `<g>` で管理 |
| 輪郭抽出 | OpenCV.js（WASM） | アルファチャンネルから輪郭ポリゴン抽出、白版のerosion |
| ポリゴン演算 | Clipper2 WASM（clipper2-js / wasm-clipper） | オフセット（ラウンドジョイン）、干渉判定、ブーリアン結合 |
| PDF書き出し | pdf-lib（低レベルAPIでOCG実装）または独自PDF生成 | レイヤー付きPDF（Optional Content Groups） |
| 状態管理 | Zustand など軽量なもの | 設定・オブジェクトリスト |

**重要原則：すべての処理はブラウザ内で完結する。外部API・サーバー処理は一切使わない。**

---

## 3. 座標系とDPI管理（最重要設計）

- **内部座標はすべてmm基準（float）で保持する。** pxはあくまで表示用・ラスター用の派生値。
- DPIは設定可能（デフォルト **350dpi**、選択肢：300 / 350 / 400、＋自由入力）。
- 変換式：`px = mm ÷ 25.4 × dpi`
- ラスター画像（カラー版・白版）は設定DPIで書き出す。
- 画面表示はズーム倍率を別途持つ（表示倍率と実寸を混同しない）。
- 丸め誤差対策：すべての幾何演算はmm（float64）で行い、px化は最終出力時の1回のみ。

## 4. キャンバス設定（新規作成時 + 設定メニューで変更可）

| 項目 | デフォルト | 選択肢 / 範囲 |
|---|---|---|
| 用紙サイズ | A4（210×297mm） | A4 / A5 / B4 / B5 / カスタム（W×H mm入力） |
| 向き | 縦 | 縦 / 横 |
| マージン（上下左右） | 5mm | 0〜20mm、上下左右個別設定可 |
| DPI | 350 | 300 / 350 / 400 / カスタム |

- マージン内側を「配置可能エリア」として点線表示。
- オブジェクトがマージンを侵食した場合は警告表示（赤ハイライト）。

## 5. 生成パラメータ（設定メニューで調整可）

| 項目 | デフォルト | 範囲 | 説明 |
|---|---|---|---|
| カットラインオフセット | 0.5mm | 0.1〜3.0mm（0.1刻み） | 画像輪郭からの距離 |
| 角丸め半径 | 0.5mm | 0.1〜2.0mm | 鋭角防止のスムージング強度 |
| オブジェクト最小間隔 | 4.0mm | 2.0〜10.0mm | カットライン同士の最短距離 |
| 白版縮小量 | 2px | 0〜10px（またはmm指定モード） | カラー版からのerosion量 |
| 輪郭の穴の扱い | 無視（外周のみ） | 外周のみ / 穴も含む | ドーナツ状カットへの対応 |

- 設定はlocalStorageに保存し、次回起動時に復元（プリセット保存機能：「いつもの入稿先A」など複数保存可）。

## 6. オブジェクト機能

### 6.1 画像オブジェクト
- 透過PNGをドラッグ＆ドロップまたはファイル選択で追加。
- 追加時に自動実行されるパイプライン：
  1. アルファチャンネルから輪郭抽出（OpenCV.js `findContours`、閾値：alpha > 0）
  2. 輪郭を簡略化（Douglas-Peucker、許容誤差 0.05mm 程度）
  3. Clipper2でオフセット（`JoinType: Round`、`delta = カットラインオフセット`）→ 凸角は自動で円弧化
  4. 凹部のスムージング：クロージング処理（+角丸め半径 → −角丸め半径 のダブルオフセット）
  5. 白版生成：アルファマスクにerosion（設定px分）をかけたマスク画像を保持
- 画像の元解像度と配置サイズからの実効DPIを表示し、設定DPI未満なら警告（例：「この画像は実効280dpiです」）。

### 6.2 操作
- **移動**：ドラッグ。矢印キーで0.1mm単位の微調整（Shiftで1mm）。
- **回転**：選択時に回転ハンドル表示。自由角度＋Shiftで15°スナップ。数値入力も可。
- **拡縮**：縦横比固定で拡縮可（拡縮時はカットライン・白版を再計算）。
- **複製**：Ctrl/Cmd+D。同じ画像の複製はパイプライン再計算をスキップ（結果を共有）。
- **削除**：Delete / Backspace。
- **Undo/Redo**：Ctrl+Z / Ctrl+Shift+Z（操作は即時実行＋Undoで戻す方針。確認ダイアログは出さない）。

### 6.3 間隔チェック（リアルタイム）
- 判定方法：各カットラインを `最小間隔 ÷ 2` だけ膨らませた判定用ポリゴンを内部保持し、判定用ポリゴン同士の交差を検出。
- ドラッグ中にリアルタイム判定し、違反ペアのカットラインを**赤色表示**。
- キャンバス外周・マージンとの干渉も同様に判定。
- 書き出し時に違反が残っている場合は警告ダイアログ（書き出し自体は可能。ここだけは例外的に確認を挟む）。

### 6.4 スタンドパーツ（ポッチ・台座）
- **組み込みパーツライブラリ**としてサイドパネルに表示。入稿先テンプレート由来の確定寸法パーツをSVGパスデータで内蔵する。
  - パーツデータの取り込み方法：Illustratorで各パーツをアートボードぴったりでSVG書き出し → `assets/parts/*.svg` として同梱 → ビルド時にmm寸法のパスデータへ変換。
  - パーツごとのメタ情報：名称、対応サイズ（例：台座 小/中/大）、凸タブ寸法、対応する台座穴寸法。
- 凸タブ（ポッチ）：本体オブジェクトにドラッグで吸着配置 → カットラインと**ブーリアン結合（union）**して一体化。タブの位置は本体輪郭に沿ってスライド調整可。
- 台座：独立オブジェクトとして配置（穴付きカットライン）。台座は白版・カラー版なし（カットラインのみ）でも、印刷ありでも選べる。
- タブと台座穴のペア整合性表示：「タブM ↔ 穴M」のように対応サイズをラベル表示。

## 7. レイヤー構成と書き出し

### 7.1 レイヤー（3層固定）

| レイヤー | デフォルト名（変更可） | 内容 | 書き出し形式 |
|---|---|---|---|
| カラー版 | `print`（※実際のデフォルト名は入稿先指定に合わせて設定画面で変更） | 配置済み透過PNG | ラスター（設定DPI） |
| カットライン | `cut` | オフセット済みパス | ベクター（ストローク。塗りなし） |
| 白版 | `white` | erosion済みマスク | ラスター。**塗り色は #000000（C0M0Y0K100相当の黒）で出力** |

- レイヤー名は設定メニューで編集可。デフォルト値はプリセットとして保存可能（入稿先ごとに切替）。
- エディタ上の表示：白版はそのままだと見えないので、**編集画面では半透明の水色などで可視化**し、書き出し時のみ #000000 に変換する。レイヤーごとの表示ON/OFFトグルあり。

### 7.2 書き出し形式

**メイン：レイヤー付きPDF（PDF 1.5+ / OCG使用）**
- 1ファイルに3レイヤー（Optional Content Groups）を格納。Illustratorで開くとレイヤーパネルに3層が表示される状態を目指す。
- カットラインはベクターパスのまま格納（線幅 0.1mm 程度、色は入稿先指定に合わせて設定可。デフォルト：シアン100%相当の青）。
- カラー版・白版は設定DPIのラスターとして埋め込み。

**サブ（保険）：SVG書き出し**
- 3つの名前付き `<g>`（レイヤー名がグループ名）で構成された単一SVG。
- Illustratorで開いてレイヤー化するための変換スクリプト（.jsx）を同梱・ダウンロード可能にする。
- OCG実装が難航した場合のv1リリース時フォールバックとしても機能する。

**その他**
- プロジェクト保存：配置情報＋設定をJSONで保存/読込（画像はbase64埋め込みまたは再リンク）。

## 9. 処理パイプライン図

```
透過PNG投入
   │
   ├─→ [輪郭抽出] アルファ→輪郭抽出 → 簡略化
   │        │
   │        └─→ [Clipper] +0.5mmオフセット(Round)
   │                 → クロージングで凹部スムージング
   │                 → カットラインパス（mm座標）
   │
   ├─→ [erosion] アルファerosion(-2px) → 白版マスク
   │
   └─→ 元画像 → カラー版

配置・回転・タブ結合（ユーザー操作）
   │
   └─→ [Clipper] 膨張ポリゴン同士の交差判定 → 間隔チェック

書き出し
   │
   └─→ [pdf-lib + OCG] 3レイヤーPDF
        ├ print: ラスター(設定DPI)
        ├ cut:   ベクターパス
        └ white: ラスター(#000000)
```

## 11. リスクと対策

| リスク | 対策 |
|---|---|
| PDFのOCG実装が複雑 | Phase 4のSVG＋jsxを先に完成させ、リリースブロッカーにしない |
| 輪郭が複雑な画像（髪の毛先など）でカットラインがギザつく | 簡略化の許容誤差と角丸め半径をパラメータ化済み。プレビューで即確認できるUI |
| 大判キャンバス＋高DPIでメモリ逼迫 | 編集中は低解像度プロキシ表示、書き出し時のみフル解像度処理 |
| Clipper2のWASMロード失敗環境 | js-angusj-clipper等の純JS実装をフォールバックに |
| 入稿先ごとの仕様差 | プリセット機能（パラメータ＋レイヤー名のセット保存）で吸収 |

## 12. 用意してもらうもの（きなこちゃん側）

1. **スタンドパーツのテンプレデータ**：Illustratorから各パーツ（凸タブ各サイズ、台座各サイズ）を個別SVG書き出し。アートボードはパーツぴったりに。
2. **入稿先の正式レイヤー名**（カラー/カット/白版それぞれ）→ デフォルト値として組み込む。
3. **カットラインの指定色・線幅**（入稿先規定があれば）。
4. テスト用の透過PNG数枚（単純な形状＋髪の毛など複雑な形状）。
